## 1. 작업 개요
- **작업명**: 공통 영역 푸터 뉴스레터 소식받기 팝업 및 신청 완료 팝업 구현

## 2. 세부 구현 사항
- **마크업 (`includes/popups/newsletter.html`)**:
  - 피그마 Node ID `2405:105383` (소식받기 팝업)과 `2405:105407` (완료 팝업) 사양을 통합한 단일 모달 마크업 설계.
  - Step 1(동의화면)과 Step 2(완료화면) 영역을 구분하여 DOM 활성화 전환이 용이하도록 구성.
- **푸터 연동 (`includes/footer.html`)**:
  - 푸터 하단에 `<load ="/includes/popups/newsletter.html" />`를 적용해 모든 화면에서 자동으로 모달 마크업을 인클루드.
  - 소식받기 입력란과 버튼에 각각 `id="input-footer-email"`, `id="btn-footer-newsletter"`를 추가.
- **스타일 (`src/css/components/modal.css`)**:
  - 모달 공통 오버레이 스타일 및 뉴스레터 전용 스타일 신규 작성.
  - 피그마 폰트 사양(NanumSquareNeo 24px weight 800, Pretendard 18px) 매핑.
  - 피그마 컬러 사양 반영: 배경색 `#fffde9`, 약관 동의 박스 배경색 `rgba(208, 201, 255, 0.2)` (불투명도 20% 보라색), 테두리 `#aaaaaa`, 필수 체크박스 테두리 `#8f8f8f` 20x20 크기, CTA 버튼 빨간색 `#db0007` (가로 150px x 세로 52px, radius 8px).
  - 모바일 반응형 처리 완료.
  - 글로벌 스타일 `src/css/style.css` 에 `@import "components/modal.css";` 연동.
- **스크립트 (`src/js/components/newsletter.js` 및 `src/js/common.js`)**:
  - 푸터에서 소식받기 클릭 시 이메일 유효성을 1차 검사 (비어있거나 잘못된 형식일 때 alert 안내 및 포커싱).
  - 이메일 유효 시 모달 활성화 및 1단계 동의 화면으로 초기화.
  - 필수 동의 체크박스가 체크되어 있을 때만 "구독 완료하기" 버튼이 활성화(disabled 해제)되며, 클릭 시 완료(Step 2) 화면으로 전환.
  - 완료 화면에서 "확인" 또는 "닫기(X)" 버튼 클릭 시 팝업이 닫히며 이메일 필드, 동의 체크 상태, 화면 단계가 모두 원래 상태로 초기화.
